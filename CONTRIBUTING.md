## 中文

感谢你考虑为 ShipAny Module Master 做出贡献！

### 如何贡献

#### 报告 Bug

1. 检查 [Issues](../../issues) 确保 bug 未被报告
2. 创建新 Issue，包含：
   - 清晰的标题
   - 详细的问题描述
   - 复现步骤
   - 预期行为 vs 实际行为
   - 环境信息（Claude 版本、操作系统等）

#### 建议新功能

1. 创建 Issue 描述功能需求
2. 说明使用场景和价值
3. 等待社区讨论和反馈

#### 提交代码

1. Fork 仓库
2. 创建特性分支：`git checkout -b feature/amazing-feature`
3. 提交更改：`git commit -m 'Add amazing feature'`
4. 推送分支：`git push origin feature/amazing-feature`
5. 创建 Pull Request

### 代码规范

- Python 代码遵循 PEP 8
- Markdown 文件使用清晰的标题层级
- 提交信息使用英文，格式：`类型: 简短描述`
  - 类型：`feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

### 文档贡献

- 修正错别字和语法错误
- 改进说明和示例
- 添加翻译（欢迎更多语言）
- 更新 ShipAny 模块信息

## English

Thank you for considering contributing to ShipAny Module Master!

### How to Contribute

#### Report Bugs

1. Check [Issues](../../issues) to ensure bug hasn't been reported
2. Create new Issue with:
   - Clear title
   - Detailed description
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment info (Claude version, OS, etc.)

#### Suggest Features

1. Create Issue describing feature request
2. Explain use cases and value
3. Wait for community discussion

#### Submit Code

1. Fork repository
2. Create feature branch: `git checkout -b feature/amazing-feature`
3. Commit changes: `git commit -m 'Add amazing feature'`
4. Push branch: `git push origin feature/amazing-feature`
5. Create Pull Request

### Code Standards

- Python code follows PEP 8
- Markdown files use clear heading hierarchy
- Commit messages in English, format: `type: brief description`
  - Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

### Documentation Contributions

- Fix typos and grammar
- Improve explanations and examples
- Add translations (more languages welcome)
- Update ShipAny module information
